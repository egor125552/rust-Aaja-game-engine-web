# Aaja Audio Engine 0.1.0 — release notes draft

## Highlights

- Rust/WebAssembly scene core with finite-value validation, distance math, room preset normalization, and deterministic voice eviction.
- ESM TypeScript API backed by browser Web Audio for cached and streaming playback, HRTF/equal-power spatialization, movement, distance, source cones, rooms, reverb, occlusion, categories, ducking, limits, and cleanup.
- Accessible main listening tour with real CC0 wooden footsteps, a real CC0 door-close transient, and a synthetic diagnostic control.
- Separate dry front/back page with equal conditions, eight angles, repeat, rating controls, and HRTF/equal-power A/B.
- Separate benchmark for up to 128 requested sources, repeated cleanup cycles, objective timing, eviction visibility, diagnostics snapshots, and JSON/text export.
- Clean npm tarball verification through strict TypeScript, direct browser ESM, Vite production build, emitted WASM, and installed-package browser runtime checks.

## HRTF limitation

The engine uses the browser's standard `PannerNode` HRTF. User listening confirmed a strong left/right effect, but front/back was weak or uncertain. Version 0.1.0 does not claim universal front/back localization and does not enable an artificial accessibility cue by default.

## Packaging

Candidate package name: `@aaja/audio-game-engine`.

The tarball is intended to contain:

- `dist/index.js`;
- TypeScript declarations;
- generated WASM JavaScript loader;
- `.wasm` binary;
- package README and metadata.

Demo assets, tests, source files, and workspace-only files must not be included.

## Publication gate

Do not publish until all of the following are confirmed:

1. final CI succeeds after the last code and documentation change;
2. package name and `@aaja` scope ownership are confirmed;
3. npm authentication is available;
4. the user authorizes the irreversible publish;
5. tag `v0.1.0` and GitHub Release are created from the verified commit.

Expected publish command after those gates:

```bash
npm publish --workspace packages/engine --access public --provenance
```

Until then, the verified tarball is the release candidate and npm publication remains external.
