# iPhone and VoiceOver checklist

Record iPhone model, iOS version, Safari version, headphones, and date before starting. Use `не указано` for anything unknown.

1. Open the GitHub Pages listening lab.
2. Turn VoiceOver on.
3. Find and activate the sound-enable button.
4. Run left and right.
5. Open the focused front/back page and run front/rear.
6. Compare HRTF and equal-power.
7. Return to the main page and switch rooms.
8. Change occlusion.
9. Run multiple sources.
10. Run speech ducking.
11. Open benchmark and run 8 sources.
12. Run 16 sources.
13. Run 32 sources.
14. Background Safari, then return.
15. Lock the phone, unlock it, and return.
16. Activate sound again if Safari suspended it.
17. Stop and restart the engine several times.
18. Use emergency stop.
19. Confirm focus does not jump after buttons or status changes.
20. Export the plain-text benchmark report.

For every failed step, record the page, control, browser state, VoiceOver state, and whether sound was silent, delayed, duplicated, distorted, or incorrectly positioned.

Do not mark this checklist as agent-tested unless a physical iPhone and VoiceOver session was actually performed.
